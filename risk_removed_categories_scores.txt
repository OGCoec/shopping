去掉类目：6 类 与 8-11 类
说明：以下仅保留 1-5、7 类；为防守侧风控设计示例分值（用于你自己的系统），不是任何平台真实规则。

==============================
1 类（网络与 IP 层）
==============================
1.1 IP 信誉低
- 加分（风险上升）：+35
- 减分（风险下降）：-15（同 IP 历史长期正常）

1.2 ASN 属于数据中心/代理网络
- 加分：+20
- 减分：-8（网络长期稳定且无异常）

1.3 同 IP 登录相关接口短时突发
- 加分：+25
- 减分：-10（速率恢复正常并持续稳定）

1.4 短时间地理位置跳变不合理
- 加分：+15
- 减分：-8（地理位置恢复稳定）

1.5 同 IP 多账号轮转
- 加分：+22
- 减分：-10（轮转行为消失）


==============================
2 类（设备与浏览器指纹层）
==============================
2.1 指纹短周期高频漂移
- 加分：+18
- 减分：-10（指纹恢复稳定）

2.2 同设备关联账号数异常
- 加分：+22
- 减分：-10（关联关系回归正常）

2.3 命中自动化环境特征（webdriver/headless 等）
- 加分：+20
- 减分：-12（特征消失且行为正常）

2.4 新设备 + 高风险网络 + 新账号叠加
- 加分：+20
- 减分：-10（通过强验证并进入观察期）

2.5 可信设备近期成功登录
- 加分：0
- 减分：-20


==============================
3 类（账号与身份层）
==============================
3.1 账号过新且触发敏感操作
- 加分：+15
- 减分：-8（账号年龄增长且行为稳定）

3.2 一次性/高风险邮箱域名
- 加分：+20
- 减分：-10（更换并验证为低风险邮箱）

3.3 手机号疑似 VoIP/虚拟号
- 加分：+20
- 减分：-10（补充稳定验证后）

3.4 同手机号或同恢复信息关联账号过多
- 加分：+16
- 减分：-8（关系收敛）

3.5 账号资料长期稳定且已验证
- 加分：0
- 减分：-12


==============================
4 类（认证流程层）
==============================
4.1 密码登录失败短时过高
- 加分：+22
- 减分：-10（失败率恢复正常）

4.2 OTP 重发频率异常
- 加分：+18
- 减分：-8（重发恢复正常）

4.3 OAuth state/nonce 校验失败
- 加分：+25
- 减分：-12（后续校验连续通过）

4.4 CAPTCHA/挑战失败
- 加分：+20
- 减分：-10（挑战通过）

4.5 MFA/CAPTCHA 强验证通过
- 加分：0
- 减分：-18


==============================
5 类（行为轨迹层）
==============================
5.1 操作节奏高度机械化
- 加分：+12
- 减分：-6（节奏恢复自然）

5.2 关键表单填充速度异常
- 加分：+10
- 减分：-5（输入行为恢复正常）

5.3 访问路径偏离正常漏斗
- 加分：+10
- 减分：-5（路径恢复正常漏斗）

5.4 同模板行为短时高重复
- 加分：+12
- 减分：-6（重复度下降）

5.5 正常人工交互节奏
- 加分：0
- 减分：-8


==============================
7 类（关联图谱层）
==============================
7.1 与高风险团簇强关联
- 加分：+18
- 减分：-10（观察期内关联风险下降）

7.2 同恢复邮箱/手机号关联账号过多
- 加分：+15
- 减分：-8（关联数回落）

7.3 同支付工具关联账号异常（若你业务有支付）
- 加分：+16
- 减分：-8（关系稳定后）

7.4 与高风险团簇距离远且关系稳定
- 加分：0
- 减分：-10


==============================
补充规则（建议）
==============================
A) 单条强信号上限：+35
B) 弱信号单条上限：+5，且不能单独触发高风险处置
C) 分值总范围：0~100
D) 分值衰减：每 24 小时衰减 30%
E) 建议处置：低分放行，中分挑战，高分冷却/临时阻断（避免一刀切）
